# Research: macOS-Native Privacy Features for DockLock (Vigil Screen) v0.5.0

## Summary

DockLock already owns the high-drama privacy gestures (instant panic, biometric proximity, Vision-based shoulder-surfing, screen-share detection). The next growth layer sits in three directions: persistent monitoring and audit transparency (logging who accessed camera/mic/clipboard and when), context-aware automatic protection (WiFi geofencing, inactivity timeouts, USB insertion triggers), and screen-content hardening (preventing other apps from capturing DockLock-managed windows and detecting passive clipboard snooping). macOS 15 Sequoia and macOS 26 Tahoe each add targeted APIs that make these features achievable without third-party dependencies — most critically, new `NSPasteboard.accessBehavior` clipboard-detection APIs (macOS 26), the `Foundation Models` on-device AI framework for local privacy intelligence, and hardened `ScreenCaptureKit` permission flows that DockLock can detect and react to. Network Extension content filtering is the one high-value area that is gated by an Apple-controlled entitlement and is not realistic for a solo indie dev without an approved request.

---

## Key Findings — Feature Candidates, Ranked by Estimated User Impact

---

### 1. Clipboard Sentinel — Monitor and Alert on Unauthorized Clipboard Reads

**What it does for the user:** Journalists drafting sensitive stories, lawyers with privileged content, and traders with price-sensitive data routinely copy secrets to the clipboard and forget them there. No existing macOS utility in the category alerts in real time when a _different_ app reads the clipboard. DockLock could show a banner ("Signal read your clipboard 3 seconds ago") and optionally auto-clear after a configurable timeout or immediately on Panic Mode.

**Which API enables it:**
- `NSPasteboard.accessBehavior` — a new read-only property (introduced as a developer preview in macOS 15.4, shipping fully in macOS 26 Tahoe) that reports whether programmatic clipboard access is always allowed, never allowed, or prompt-gating.
- New `detect` methods on `NSPasteboard` and `NSPasteboardItem` allow inspecting what _types_ of data are on the pasteboard without triggering the alert itself, letting DockLock make smart decisions about whether to auto-clear.
- `NSPasteboard.changeCount` polling (already public, available since macOS 10.0) can detect clipboard modifications from any process.

**Competitive gap:** OverSight monitors camera/mic. Micro Snitch monitors camera/mic. No shipping macOS utility in the "privacy sentinel" category monitors _clipboard read events_ with process attribution and reactive clearing. DockLock already implements clipboard-clear on Panic Mode; this extends it to passive continuous monitoring.

**Implementation complexity: Low.** `changeCount` polling requires no entitlement. The new `accessBehavior` property requires no special entitlement. Full process attribution may require `Accessibility` permission. v0.5.0 scope: detect read/write events, show notification, offer auto-clear after N seconds.

---

### 2. Inactivity Lock — Custom Idle-Timeout Panic Trigger

**What it does for the user:** Remote workers in coffee shops and open offices want the screen to go private after 30–90 seconds of inactivity without waiting for macOS's coarser system screensaver timeout. Today DockLock has Proximity Lock (Bluetooth RSSI) and Shoulder Surfing (camera), but nothing fires when the user simply walks away. An idle-timeout feature triggers Panic Mode or a softer blur overlay after a configurable inactivity window.

**Which API enables it:**
- `IOHIDGetIdleTime` via `IOKit` — reads the system-level idle time in seconds with no permission required.
- `NSEvent.addGlobalMonitorForEvents(matching:)` — detects mouse moves and clicks globally (requires Accessibility permission), resetting the inactivity counter on activity.
- `DistributedNotificationCenter` — listen for `com.apple.screenIsLocked` / `com.apple.screenIsUnlocked` to avoid double-triggering when the system already locks.

**Competitive gap:** Amphetamine prevents sleep but does not trigger privacy actions. Idle timeout is orthogonal to Proximity Lock — it fires when the user just forgets, in a location where BLE beacons are not a reliable signal.

**Implementation complexity: Low.** `IOHIDGetIdleTime` is a single C IOKit call wrapped in a `Timer`. Only subtlety is not double-triggering with existing Proximity Lock state.

---

### 3. WiFi Geofencing — Trusted Network Auto-Lock / Auto-Unlock

**What it does for the user:** A lawyer leaves their office WiFi (trusted) and connects to a hotel or conference WiFi (untrusted). DockLock automatically enters a hardened profile: Panic Mode armed, Shoulder Surfing enabled, clipboard auto-clear on. When they return to the office network, it relaxes automatically. This is the spatial analogue of Proximity Lock — no phone or AirTag required.

**Which API enables it:**
- `CoreWLAN` — `CWWiFiClient.shared().interface()?.ssid()` returns the current SSID. Observe `CWWiFiClientDidUpdateSSIDNotification` for network changes. Requires `com.apple.developer.networking.wifi-info` entitlement and Location Services permission.

**Competitive gap:** No macOS menu-bar privacy app in the category implements SSID-based security profiles. Enterprise MDM tools (Jamf + Senturo) do geofencing but require fleet management.

**Implementation complexity: Medium.** The entitlement is self-serviceable via Xcode. The design challenge is false-positive handling (SSID spoofing) and graceful behavior when Location is denied.

---

### 4. Microphone / Camera Access Monitor (OverSight Parity + Reactive)

**What it does for the user:** When another app activates the microphone or camera while DockLock is running, the user sees an alert with the offending process name. Unlike OverSight (notification only), DockLock makes it _actionable_: one click triggers Panic Mode, adds the app to a block list, or forces mute.

**Which API enables it:**
- `AVCaptureDevice` + KVO on `isInUseByAnotherApplication` — detects whether camera/mic is in use by any other process. No entitlement required beyond Camera/Microphone usage descriptions.
- Process attribution via `NSWorkspace.shared.runningApplications` heuristic.

**Competitive gap:** OverSight is a separate app with no reactive integration. Micro Snitch requires a paid subscription. No privacy mode app integrates camera/mic detection as an _arming input_ for a broader response.

**Implementation complexity: Low-Medium.** Detection is low; configurable allowlist + reactive Panic Mode trigger is medium.

---

### 5. Screen-Recording Detection Enhancement — Passive Mode Hardening

**What it does for the user:** Extend Presentation Mode to detect when any process is _actively recording_ (not just sharing), alert the user, and optionally trigger Panic Mode.

**Which API enables it:**
- `SCContentSharingPickerObserver` (ScreenCaptureKit, macOS 14+) — notifies when external processes request capture via the picker.
- `CGWindowList` with `kCGWindowSharingState` key — DockLock's existing Presentation Mode approach; extend to also flag recording.
- `NSWindowSharingNone` (`window.sharingType = .none`) — **broken on macOS 15.4+**: ScreenCaptureKit ignores this flag. Ship as best-effort only with clear labeling.

**Implementation complexity: Medium.** Reliable detection is the hard part. CGWindowList + SCContentSharingPickerObserver combination gives reasonable coverage.

---

### 6. Privacy Audit Log — Local, Exportable Access History

**What it does for the user:** An audit log showing "Camera activated by Zoom 14:32:01, clipboard read by Bear 14:33:17, Panic Mode triggered 14:35:00" stored locally as an encrypted file gives an accountability trail no other privacy app in this category provides.

**Which API enables it:**
- Aggregate all existing signals (clipboard changeCount, `AVCaptureDevice.isInUseByAnotherApplication`, screen lock notifications, `NSWorkspace.didActivateApplicationNotification`) with timestamps.
- `NSFileManager` with `NSFileProtectionComplete` attributes for APFS encryption at rest.
- Optional: macOS 26 `Foundation Models` (on-device LLM, Apple Silicon only) for natural-language "Privacy Summary" — "3 suspicious clipboard reads during your 2-hour session."

**Competitive gap:** No consumer-facing macOS privacy app produces a per-session audit log with mixed camera/clipboard/lock events.

**Implementation complexity: Low-Medium.** Signal aggregation is low; Foundation Models natural-language summary is medium (macOS 26 / Apple Silicon only, must degrade gracefully).

---

### 7. USB / Removable Storage Alert — Trigger on Unauthorized Media Insertion

**What it does for the user:** Immediate alert when an unknown USB drive is connected; optionally trigger Panic Mode. Prevents drive-by exfiltration in shared physical spaces.

**Which API enables it:**
- `NSWorkspace.didMountNotification` — fires immediately when any volume mounts. No entitlement required.
- `DiskArbitration` framework — for vendor IDs and removable-volume detection.
- `IOKit` USB matching — for VendorID/ProductID allowlisting.

**Competitive gap:** No macOS privacy menu-bar app monitors USB insertion as a security signal.

**Implementation complexity: Low.** `didMountNotification` is a two-line subscription.

---

### 8. Head-Pose Gaze Direction (Vision Framework Extension)

**What it does for the user:** If the _user themselves_ is looking away from the screen (yaw > 30°, sustained 5 seconds), start a countdown to blur. Softer privacy tier before Panic Mode fires.

**Which API enables it:**
- `VNFaceObservation.yaw` / `.pitch` / `.roll` — already available in Vision framework; DockLock already has the camera pipeline.
- No new entitlement needed.

**Implementation complexity: Medium.** Vision pipeline already exists; the addition is reading `yaw` and implementing a sustained-absence timer. UX calibration to avoid false positives on natural head turns is the real work.

---

### 9. Anti-Screenshot Window Mode (Best-Effort Only)

**Critical caveat:** `NSWindow.sharingType = .none` is broken on macOS 15.4+ — ScreenCaptureKit ignores it and captures the window anyway. Ship as best-effort with clear in-app labeling. Do not promise as a security guarantee.

**Implementation complexity: Low to implement, but integrity is compromised.**

---

## Trade-offs / Caveats

- **Clipboard process attribution:** `changeCount` tells you the clipboard changed; it doesn't identify the reading process. Full attribution requires Accessibility permission. macOS 26 `accessBehavior` doesn't expose process names to third-party observers.
- **`NSPasteboard.accessBehavior` timeline:** Developer preview in macOS 15.4 (May 2025), fully in macOS 26 Tahoe, but reportedly disabled by default in 26.0 — may require `defaults write`. Plan for macOS 26.1+ full availability.
- **`NSWindowSharingNone` broken on macOS 15.4+:** Multiple independent developer reports confirm ScreenCaptureKit bypasses this flag (Tauri #14200, Apple Developer Forums #792152).
- **CoreWLAN SSID requires Location permission:** Since macOS Ventura, reading SSID requires `com.apple.developer.networking.wifi-info` entitlement + Location Services. Adds a non-obvious permission prompt.
- **Network Extension (content filtering) is entitlement-gated:** Requires explicit Apple approval — not v0.5.0 material.
- **Foundation Models:** Apple Silicon + macOS 26 only. Must degrade gracefully on Intel.
- **OverSight is open-source:** Review its source for edge cases before building Feature #4.

## Recommended v0.5.0 Sequence (Impact × Feasibility)

| Priority | Feature | Complexity | Killer Differentiator |
|---|---|---|---|
| 1 | Inactivity Lock (idle timeout) | Low | Fills gap left by Proximity Lock in no-BLE environments |
| 2 | Clipboard Sentinel (monitor + auto-clear) | Low | No existing tool does this |
| 3 | USB / Removable Media Alert | Low | Unique in the category, zero entitlement |
| 4 | Privacy Audit Log (local JSON) | Low-Medium | Unique in the consumer privacy app space |
| 5 | WiFi Geofencing (trusted SSID) | Medium | Unique; needs Location permission handling |
| 6 | Camera/Mic Integration (OverSight-parity + reactive) | Low-Medium | Parity + reactive advantage over OverSight |
| 7 | Head-Pose Gaze Away Detection | Medium | Extends existing Vision pipeline; novel |
| 8 | Foundation Models Privacy Summary | Medium | macOS 26 / Apple Silicon only; degrade gracefully |
| 9 | Anti-Screenshot Window Mode | Low (best-effort) | Ship as clearly labeled best-effort only |
